## 업데이트 규칙

### 업데이트 트리거
1. **작업 완료 시** - 에이전트 작업이 끝나면 담당 문서 업데이트
2. **중요 결정 시** - 설계 결정이 내려지면 관련 문서에 반영
3. **사용자 요청 시** - "문서 업데이트" 명령 시 실행

### 업데이트 방식
1. **기존 문서 수정** - 해당 섹션 찾아서 내용 추가/수정
2. **변경 로그 기록** - `01. docs/changelog.md`에 변경 이력 추가

### 변경 로그 형식
```markdown
## [날짜] - [에이전트명]

### 변경 내용
- [문서명]: [변경 설명]

### 관련 결정
- [결정 내용]
```

## 문서 생성 규칙

### 새 문서가 필요한 경우
1. 새로운 콘텐츠 유형 추가 시 (새 클래스, 새 시스템 등)
2. 새로운 화면/기능 추가 시
3. 기존 문서가 너무 커질 때 분리

### 문서 생성 시 필수 사항
- 적절한 폴더에 배치
- 이 매핑 파일에 추가
- 변경 로그에 생성 기록

## 연관 문서 관리

문서 간 의존성을 관리하여 연관 문서가 함께 업데이트되도록 합니다.

### 의존성 맵
- **경로**: `.claude/docs/dependency-map.md`
- **용도**: 문서 간 연관 관계 정의
- **사용 시점**: 문서 업데이트 시 연관 문서 파악

### 연관 문서 업데이트 절차
```
1. 문서 수정 완료
2. dependency-map.md에서 연관 문서 확인
3. 연관 문서 업데이트 필요 여부 검토
4. changelog.md 필수 업데이트
```

> 상세 정보: `.claude/docs/dependency-map.md`
